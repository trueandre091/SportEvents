# Платформа для взаимодействия представителей Федерации спортивного программирования

Веб-платформа, разработанная для улучшения координации между центральными и региональными представителями ФСП. Система обеспечивает эффективную коммуникацию, упрощает процессы планирования мероприятий и принятия решений.

## Проблема и решение

Федерации спортивного программирования необходимо поддерживать постоянную коммуникацию между центральными и региональными представителями. Из-за отсутствия налаженных процессов создается избыточная нагрузка на сотрудников, замедляется принятие решений, усложняется планирование мероприятий.

Наша платформа решает эти проблемы, предоставляя единое пространство для взаимодействия всех участников процесса.

## Ключевые особенности

### 1. Автоматизированная система доступа
- Автоматическая рассылка логинов и паролей представителям ФСП
- Мгновенный доступ к системе без длительного процесса регистрации
- Контролируемое создание учетных записей для уполномоченных лиц

### 2. Система уведомлений
- Интеграция с Telegram для мгновенных уведомлений
- Настраиваемые email-оповещения
- Напоминания о предстоящих событиях

### 3. Автоматизированный архив событий
- Автоматическое и ручное обновление базы прошедших мероприятий
- Доступ к историческим данным
- Удобный поиск по архиву
- Официальная документация всех проведенных мероприятий

## Роли пользователей и их возможности

### Представитель центральной ФСП
- Просмотр всех событий с фильтрацией, поиском, возможностью добавления, изменения, удаления
- Добавление, изменение, удаление представителей и пользователей
- Доступ к общей статистике по периодам с возможностью экспорта в Excel

### Представитель региональной ФСП
- Организация региональных событий
- Создание и изменение событий с возможностью добавления документов
- Доступ к общей статистике по периодам с возможностью экспорта в Excel

### Пользователь
- Регистрация через email
- Подключение Telegram для уведомлений
- Подписка на интересующие события
- Получение уведомлений о мероприятиях
- Доступ к информации о событиях

## Как начать работу

## Особенности регистрации и входа
- 

### Для пользователей
1. Зарегистрируйтесь на платформе
2. Получите Telegram ID через бота @LeadConverterToolkitBot (опционально)
3. Перейдите во вкладку "События"
4. Подпишитесь на интересующие мероприятия
5. Во вкладке "Подписки" вы можете отписаться или настроить уведомления

### Для представителей ФСП
- Дождитесь получения автоматического письма с доступом
- Или обратитесь к администратору для ручного создания учетной записи
- Во вкладке "Admin Панель" вы можете добавлять, удалять, изменять события и пользователей
- Во вкладке "Статистика" вы можете просматривать статистику по периодам и экспортировать в Excel

## Установка и запуск

#### Требования
- Docker
- Docker Compose
- git

### Локальный запуск
#### Установка
1. Клонируйте репозиторий
2. Если у вас Windows, то установите Docker Desktop и откройте его
3. Введите команду для создания локальной сети:
    ```bash
    docker network create sportevents-network
    ```
4. Перейдите в ветку "backend"
  - Из корневой папки выполните команду:
    ```bash
    docker-compose up --build
    ```
  - Можете закрыть терминал, приложение будет запущено в фоновом режиме, проверить можно в Docker Desktop
5. Перейдите в ветку "frontend"
  - Откройте терминал и выполните команды:
    ```bash
    cd app
    docker-compose up --build
    ```
  - Приложение будет запущено в браузере, перейдите по адресу http://localhost:3000

### Запуск на сервере
1. Установить Docker и git
2. Клонировать репозиторий
3. Выполнить команды для перехода в ветку и запуска контейнера backend:
    ```bash
    cd pp_final_20529_szfo_kgd_404_95
    git checkout backend
    git pull
    docker-compose up --build
    ```
4. Выполнить команды для перехода в ветку и запуска контейнера frontend:
    ```bash
    cd pp_final_20529_szfo_kgd_404_95 # если ещё не в этой папке
    git checkout frontend
    git pull
    cd app
    docker-compose up --build
    ```
5. Приложение будет доступно по адресу http://your_server_ip:3000





### Установка зависимостей
```bash
# Установка зависимостей frontend
cd app
npm install

# Установка зависимостей backend
cd server
npm install
```

### Настройка окружения
1. Создайте файл `.env` в папке `server` со следующими параметрами:
```env
DB_HOST=localhost
DB_USER=your_username
DB_PASSWORD=your_password
DB_NAME=your_database_name
JWT_SECRET=your_jwt_secret
```

2. Создайте базу данных и выполните миграции:
```bash
cd server
npm run migrate
```

### Запуск приложения
```bash
# Запуск frontend (режим разработки)
cd app
npm start

# Запуск backend
cd server
npm start
```

## Структура проекта

```
src/
  ├── api/          # API-клиенты

  ├── context/      # React контексты
  ├── pages/        # Страницы приложения
  ├── utils/        # Вспомогательные функции
```

### Backend (/server)
```
src/
  ├── controllers/  # Контроллеры
  ├── middleware/   # Промежуточные обработчики
  ├── models/       # Модели данных
  ├── routes/       # Маршруты API
  └── app.js        # Основной файл сервера
```
